# AI 陪玩 客户端原型 · 项目说明书

> 本文档是项目的知识库主文档，面向后续接手开发的工程师，记录项目现状、已实现/未实现功能、以及功能拆分与重构建议。
> 配套文档：[`页面元素索引.md`](./页面元素索引.md)（按 id 检索页面元素的字典）。

## 1. 项目概述

- **定位**：一款「AI 陪玩」桌面客户端的 UI 原型（含本地伙伴形象、游戏启动、屏幕权限、登录注册、设置等界面）。
- **来源**：由 **Pixso** 设计稿直接导出的静态 HTML，后期手工补充了页面切换脚本。
- **技术栈**：
  - 原生 HTML + 外置 CSS（`css/styles.css`、`css/font.css`），无构建工具、无框架
  - 业务脚本 `js/app.js`（Tab 切换、列表渲染、弹窗、事件总线 `CompanionUI`）
  - 遗留 Pixso 导航脚本 `js/navigation.js`（旧版单文件导出稿用，当前主入口为 `Index.html` + `app.js`）
- **运行方式**：用浏览器打开根目录 `Index.html`；建议本地静态服务器以保证字体/图片路径正确。
- **设计稿基准尺寸**：页面高度约 `1024px`，采用绝对定位 + 像素坐标布局（非响应式）。

## 3. 架构说明

### 3.1 Pixso 导出特征
- 每个「画板/页面」导出为一个顶层 `<div class="Pixso-frame-XXX">`，彼此平铺在 `.scroll-container > .Pixso-canvas-0_1` 容器内。
- 元素 **id 命名规则为 `数字_数字`**（如 `3_51`、`79_262`），对应 Pixso 内部节点 id，**无业务语义**。
- 元素定位基本都是 `position: absolute` + 固定 `left/top/width/height`。
- 文本节点是 `<p>` / `<span>`，图标多为 `<div class="Pixso-vector-*">` 配合背景图或 `stroke-wrapper`。

### 3.2 页面切换机制
没有路由，纯靠 **显隐切换**实现"多页面"：
- 初始化时把所有非当前页面 `display:none`（`navigation.js` 的 `INIT_HIDDEN`）。
- 切换主 Tab 时，显示目标 frame、隐藏其它主 frame，并隐藏所有浮层（`OVERLAY_PAGES`）。
- 显示页面时强制设置 `position/top/left/opacity`，覆盖 GSAP 导出脚本可能造成的误隐藏（脚本注释提到修复了"Pixso 导出后 GSAP 时间线误隐藏 Tab 按钮"的问题）。

### 3.3 对外接口
`app.js` 在 `window` 上挂载 `CompanionUI`，供调试或后续扩展：

```js
window.CompanionUI = {
  getState(),              // activeTab、activeSettingsSubtab、userProfile、generalSettings 等
  setTab(tabName),         // '首页' | '伙伴' | '游戏' | '设置'
  setSubtab(subtabName),   // 伙伴页：'模型' | '工坊'
  setSettingsSubtab(name), // 设置页：'个人资料' | '快捷键设置' | …
  saveUserProfile(),       // 保存个人资料到 state + localStorage
  openModal(name),         // 'confirm-delete' | 'add-permission' | 'game-detail'
  closeModal(),
  openGameDetail(game),    // 打开游戏详情弹窗并填充数据
  showToast(type, message),
  on / off                 // 事件总线：tab:change、settings:subtab-change、game:launch 等
};
```

旧版 `navigation.js` 仍提供 `window.AppNav`（Pixso 数字 id 页面切换），与当前 DOM 结构可能不一致，新功能请以 `app.js` 为准。

### 3.4 关键常量（位于 `navigation.js` 顶部）
| 常量 | 作用 |
|---|---|
| `PAGE` | 4 个主页面 id：home=`3_65`、partner=`3_51`、game=`17_31`、settings=`17_62` |
| `WORKSHOP_PAGE` | 工坊/模型页 id：`6_42` |
| `INIT_HIDDEN` | 初始需隐藏的所有非默认面板 id 列表 |
| `OVERLAY_PAGES` | 切主 Tab 时需一并隐藏的浮层/变体页 |
| `NAV_GROUPS` / `NAV_LABELS` | 12 套导航栏（按钮 + 文字）的 id，统一批量绑定点击 |

## 6. 功能拆分与重构建议

针对"为后续开发做功能拆分"的目标，建议分两步走：

### 6.1 业务模块划分（先逻辑拆分，不动 HTML）
把 `navigation.js` 拆成多个职责单一的模块，统一由一个入口初始化：

| 建议模块 | 职责 | 涉及页面/元素 |
|---|---|---|
| `core/pageManager` | 显隐切换、初始隐藏、`AppNav` 接口 | 所有 frame |
| `modules/nav` | 主 Tab 导航绑定 | `NAV_GROUPS` / `NAV_LABELS` |
| `modules/workshop` | 工坊/模型页、文件上传 | `6_42` |
| `modules/auth` | 登录/注册表单与校验 | `72_275` |
| `modules/settings` | 设置子栏目切换、个人资料编辑 | `17_62` 及变体 |
| `modules/games` | 游戏列表渲染、启动按钮 | `17_31`、`45_*`、`21_46` |
| `modules/permission` | 屏幕权限弹窗、窗口选择 | `50_312`、`50_257/272` |

> 当前各模块所需的 id 已集中在 `navigation.js` 顶部常量，可平移到各自模块，降低相互耦合。

### 6.2 工程化重构（中长期，可选）
1. **拆分巨型 `Index.html`**：把内联 `<style>` 抽到独立 CSS；按页面把 frame 拆到分文件或组件。
2. **id 语义化**：保留 Pixso 原 id 的同时，给关键元素加 `data-role="login-submit"` 之类语义属性，脚本改用语义选择器，避免硬编码数字 id（见 `页面元素索引.md`）。
3. **数据动态化**：游戏列表、伙伴信息、用户资料改为数据驱动渲染。
4. **真实表单**：登录/注册的静态 `<p>` 换成 `<input>`，接入校验与后端接口。
5. **引入路由/状态**：页面数量增长后，用轻量路由（hash 路由）替代纯 `display` 切换。

## 8. 开发者上手指南

1. **跑起来**：浏览器打开 `Index.html`（推荐本地静态服务器）。登录后默认 **首页** Tab。
2. **找页面**：`Index.html` 中 `#tab-首页`、`#tab-伙伴`、`#tab-游戏`、`#tab-设置`。
3. **找元素**：查 [`页面元素索引.md`](./页面元素索引.md)（语义化 id）。
4. **加交互**：在 `js/app.js` 中复用 `openModal` / `setSettingsSubtab` / `Bus.emit` 等现有模式。
5. **切 Tab 调试**：控制台 `CompanionUI.setTab('设置')`；切换设置子栏目：`CompanionUI.setSettingsSubtab('语音设置')`。
6. **样式**：设置页见 `css/styles.css` 中 `.settings-*` 区块；游戏详情弹窗见 `.modal-game-detail`。

## 9. 游戏详情弹窗（已实现）

### 9.1 交互
- **触发**：仅在 **游戏 Tab**（`#tab-游戏`）点击游戏卡片主体（不含 hover 出现的「启动」按钮）时打开。
- **关闭**：右上角 ×、底部「关闭」、点击遮罩、`Esc`（与其它弹窗共用 `closeModal`）。
- **启动**：弹窗底部「启动游戏」→ `game:launch` 事件 + 成功 Toast，并关闭弹窗。

### 9.2 布局与数据
- **左侧**：`game.coverImg`，与列表卡片同一海报资源。
- **右侧**：游戏类型（`genre`）、适宜人群（`audience`）、游戏背景（`description`）；当前为统一占位文案，定义于 `GAME_DETAIL_PLACEHOLDER`，可按游戏在 `gameLibrary` 中覆盖。
- **DOM**：`#modal-game-detail`，子元素 id 见 [`页面元素索引.md`](./页面元素索引.md) §10。

### 9.3 扩展
- 接入后端时：在 `openGameDetail` 前请求详情并 `populateGameDetailModal` 即可。
- 若首页「最近游戏」也需详情：在 `renderRecentGames` 的点击回调中调用 `openGameDetail`（当前仍为 Toast 选中提示）。
